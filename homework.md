1. 연결의 기초: Local과 Remote의 만남
 내 컴퓨터(Local)의 프로젝트를 GitHub(Remote)와 연결하는 것은 "내 작업실을 클라우드와 동기화하는 과정"입니다.

 git remote add origin [URL]: 내 컴퓨터의 현재 폴더가 GitHub의 특정 저장소를 바라보게 설정하는 명령입니다.

 왜 하는가?: 이를 통해 push나 pull을 할 때마다 매번 주소를 입력할 필요 없이, 'origin'이라는 이름으로 간편하게 통신할 수 있습니다.

2. 데이터 흐름: Push와 Pull
 이 두 가지는 개발자에게 가장 빈번하게 발생하는 '데이터 주고받기' 과정입니다.

 Push (밀어넣기):

 목적: 내 컴퓨터에서 작성한 코드를 GitHub 서버로 전송하여 안전하게 저장하고, 팀원들이 볼 수 있게 합니다.

 과정: add (스테이지에 올리기) → commit (버전 기록 생성) → push (서버로 전송) 순서로 진행됩니다.

 Pull (당겨오기):

 목적: GitHub 서버에 있는 최신 코드를 내 컴퓨터로 내려받습니다.

 중요성: 여러 명이 동시에 작업할 때, 내 컴퓨터에 있는 코드가 과거의 것이 되지 않도록 최신 상태로 유지해 줍니다. (충돌을 방지하는 첫걸음입니다.)

3. Markdown (.md) 활용: 협업의 기록
 .md 파일은 단순한 메모장이 아니라 '프로젝트의 얼굴'입니다.

 가독성: #, , - 같은 문법을 사용하면 코드를 모르는 사람도 읽기 편한 구조적인 문서(헤더, 굵게, 리스트 등)가 완성됩니다.

 GitHub의 역할: GitHub는 저장소에 있는 README.md 파일을 자동으로 인식하여 메인 화면에 띄워줍니다.

 왜 쓰는가?: 다른 사람이 내 코드를 봤을 때 "이 코드는 어떤 프로젝트인지", "어떻게 설치하고 실행하는지"를 설명해주기 위함입니다. 이를 '문서화'라고  합니다.
