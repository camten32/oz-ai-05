# Git & GitHub 학습 기록

## 1. Git과 GitHub의 기초
- **연결 (Remote 설정)**: 내 컴퓨터(Local)의 프로젝트를 GitHub(Remote)와 1:1로 연결합니다.
  - 명령어: `git remote add origin [저장소 URL]`
- **데이터 흐름**:
  - **Push (밀어넣기)**: 내 컴퓨터에서 작업한 내용을 GitHub 서버로 전송합니다.
  - **Pull (당겨오기)**: GitHub 서버의 최신 내용을 내 컴퓨터로 가져옵니다.

## 2. 작업 프로세스 (수정 및 반영)
코드를 수정하고 GitHub에 반영할 때는 항상 아래 순서를 따르는 것이 안전합니다.

1. **파일 수정 및 저장**: 원하는 대로 코드나 문서를 수정합니다.
2. **스테이징**: 수정된 파일을 반영할 준비를 합니다.
   - `git add .` (모든 변경 사항 반영)
3. **커밋**: 변경 기록을 남깁니다.
   - `git commit -m "수정 내용 요약"`
4. **최신 상태 업데이트 (권장)**: 충돌 방지를 위해 먼저 당겨옵니다.
   - `git pull origin main`
5. **푸시**: 서버로 최종 전송합니다.
   - `git push origin main`

## 3. Markdown (.md) 활용
- **목적**: 프로젝트의 구조, 실행 방법, 설명 등을 가독성 있게 기록합니다.
- **특징**: GitHub는 `.md` 파일을 자동으로 인식하여 웹상에서 예쁘게 렌더링해줍니다. 
- **주의사항**: 변경 사항은 항상 내 컴퓨터에서 수정 후 `push`하는 것이 버전 관리와 협업 측면에서 정석입니다.
